# 전세낀 분석 데이터 공유 시스템 기획서

**작성일:** 2026-03-12  
**관련 도구:** 전세낀 크롬 확장 프로그램  
**문서 목적:** 사용자 간 단지 분석 데이터 공유 아이디어 검토 및 구현 방향 정리

---

## 1. 문제 정의

### 현재 상황
- 사용자마다 **각자 로컬 PC에서** 아파트 단지 매물을 전수조사
- 같은 단지(예: 헬리오시티)를 A사용자가 이미 분석했어도, B사용자는 동일 작업을 처음부터 반복
- 690개 매물 기준 분석 시간: **약 30분** → 중복 낭비 발생

### 해결하고 싶은 것
> "이미 누군가 분석한 단지라면, 그 결과를 바로 가져다 쓸 수 있게 하자"

---

## 2. 핵심 제약 조건

| 제약 | 내용 | 대응 방향 |
|------|------|---------|
| 데이터 신선도 | 매매 매물은 매일 변함 (계약·취하·신규) | TTL 24시간 정책 |
| 개인정보 | 중개사 연락처, 구체적 매물 정보 포함 가능 | 최소 정보만 공유 (키워드 히트 여부 + 링크) |
| 인프라 비용 | 무료 또는 최소 비용 유지 | Supabase/Google Sheets 무료 티어 활용 |
| 사용자 동의 | 분석 결과를 강제로 수집하면 거부감 | Opt-in 방식 (사용자 선택) |

---

## 3. 시스템 개요

### 동작 흐름

```
[사용자가 단지 페이지 진입]
        ↓
[articleId로 공유 DB 조회]
        ↓
   ┌────┴────┐
  있음       없음
(24h 이내)    ↓
   ↓       [직접 분석 실행 (30분)]
[캐시 즉시        ↓
 반환]      [완료 후 "공유하기" 버튼 표시]
              ↓ (사용자 동의 시)
           [DB에 결과 저장]
```

### 공유 데이터 구조 (최소화 원칙)

```json
{
  "articleId": "2614236762",
  "complexName": "헬리오시티",
  "analyzedAt": "2026-03-15T16:24:00+09:00",
  "totalScanned": 690,
  "results": [
    {
      "keyword": "전세낀",
      "hitCount": 23,
      "listings": [
        {
          "naverUrl": "https://m.land.naver.com/...",
          "price": "13억",
          "area": "84㎡",
          "floor": "12층",
          "summary": "전세 4억 끼고 매수 가능"
        }
      ]
    }
  ]
}
```

> **포함하지 않을 것:** 중개사 이름, 전화번호, 개인 식별 정보

## 4. 구현 방향별 비교

### 방향 A — Google Sheets + Apps Script (1단계 권장)

**구조:**
```
전세낀 확장  →  Google Apps Script API  →  공유 Google Sheets
                                              ↑
                         다른 사용자 조회 ──────┘
```

**장점:**
- 완전 무료
- 구현 난이도 낮음 (스크립트 + API 키만 필요)
- 스프레드시트로 관리자가 데이터 직접 확인/수정 용이

**단점:**
- Apps Script 호출 제한 (분당 100회)
- 동시 접속 많으면 느려질 수 있음
- 실시간 동기화 불안정

**적합한 시점:** 사용자 수 ~수백 명 이하

---

### 방향 B — Supabase 무료 티어 (2단계 권장)

**구조:**
```
전세낀 확장  →  Supabase REST API  →  PostgreSQL DB (클라우드)
                                          ↑
                    다른 사용자 조회 ────────┘
```

**장점:**
- 무료 (월 50만 row, 500MB, API 무제한)
- 실시간 구독(Realtime) 지원
- Row Level Security로 데이터 보호 가능
- SQL 쿼리로 articleId 중복 체크 간단

**단점:**
- 초기 설정 필요 (DB 스키마, API 키 관리)
- 확장 프로그램에 API 호출 코드 추가 개발 필요

**적합한 시점:** 사용자 수 수백~수만 명

---

### 방향 C — Firebase Realtime Database

**장점:** 실시간 동기화 우수, Google 생태계  
**단점:** 무료 티어 제한 빡빡함 (1GB 저장, 10GB/월 전송)  
**결론:** Supabase 대비 메리트 없음 → **비권장**

---

### 방향 D — GitHub JSON 파일

**장점:** 완전 무료, 버전 관리 가능  
**단점:** 실시간 공유 불가, PR 프로세스 번거로움  
**결론:** "단지별 최근 스냅샷 보관" 아카이브 용도로만 적합

---

## 5. 데이터 신선도 정책

매물은 매일 변하므로 **오래된 캐시를 신뢰 가능한 데이터로 제공하면 안 됩니다.**

| 캐시 경과 시간 | 처리 방식 | UI 표시 |
|-------------|---------|--------|
| 0 ~ 6시간 | 캐시 즉시 제공 | "✅ 오늘 분석된 데이터입니다" |
| 6 ~ 24시간 | 캐시 제공 + 경고 | "⚠️ X시간 전 분석 데이터입니다. 참고용으로만 사용하세요" |
| 24시간 초과 | 캐시 제공 안 함 | "🔄 최신 데이터가 없습니다. 직접 분석을 실행해 주세요" |

## 6. 사용자 경험 시나리오

### 시나리오 1 — 신선한 캐시 있는 경우

```
1. 사용자가 헬리오시티 단지 페이지에서 전세낀 실행
2. 확장 프로그램이 DB 조회 (articleId: 2614236762)
3. 3시간 전 분석 데이터 발견
4. 팝업: "✅ 오늘 오후 1시에 분석된 데이터가 있습니다. 바로 확인하시겠어요?"
   → [바로 보기] [직접 재분석]
5. 사용자 "바로 보기" 클릭 → 30분 절약
```

### 시나리오 2 — 캐시 없는 경우

```
1. 사용자가 분석 실행 (30분 소요)
2. 완료 후 팝업: "분석이 완료되었습니다. 다른 중개사님들과 공유하시겠어요?"
   → [공유하기 (익명)] [공유 안 함]
3. 사용자 "공유하기" 선택 → DB 저장
4. 이후 24시간 내 다른 사용자가 같은 단지 분석 시 캐시 제공
```

## 7. 개인정보 보호 설계

### 공유되는 정보 (OK)
- 단지명, articleId
- 매물 가격대, 면적, 층수
- 매물 특징 텍스트 (광고 문구)
- 네이버 부동산 링크 (이미 공개된 정보)
- 분석 일시

### 공유되지 않는 정보 (차단)
- 중개사 이름, 연락처
- 사용자 식별 정보 (분석한 사람이 누구인지)
- IP 주소

### 구현 방식
- 확장 프로그램에서 **업로드 전 필터링** 처리
- 중개사 연락처 필드 제거 후 전송

## 8. 구현 단계별 로드맵

### 1단계 — 기반 작업 (즉시 가능)
- [ ] 전세낀 분석 결과 CSV에서 공유 가능한 필드 식별
- [ ] 공유 데이터 JSON 스키마 확정
- [ ] "공유하기" opt-in UI 와이어프레임 작성

### 2단계 — Google Sheets 프로토타입 (1~2주)
- [ ] 공유용 Google Sheets 생성 (컬럼: articleId, complexName, analyzedAt, data_json, ...)
- [ ] Apps Script API 엔드포인트 작성 (GET 조회 / POST 저장)
- [ ] 전세낀 확장 프로그램에 API 호출 로직 추가
- [ ] TTL 체크 로직 구현

### 3단계 — Supabase 마이그레이션 (사용자 증가 시)
- [ ] Supabase 프로젝트 생성
- [ ] `complex_analyses` 테이블 설계
- [ ] Google Sheets 데이터 마이그레이션
- [ ] 확장 프로그램 API 엔드포인트 교체

### 4단계 — 고도화 (선택)
- [ ] 단지별 "분석 히스토리" 페이지 (시계열 트렌드 확인)
- [ ] 특정 단지 알림 구독 (새 분석 업로드 시 알림)
- [ ] 공식 사이트(GitHub Pages)에 인기 단지 분석 결과 공개 페이지 추가

## 9. 기술 스택 요약

```
[크롬 확장 프로그램]
    ↕ REST API (JSON)

    ↕
[공유 DB]
    ↕ (3단계 이후)
[GitHub Pages 공개 페이지]
    → 인기 단지 최신 분석 결과 공개
    → SEO 효과 (단지명 + 전세낀 키워드 색인)
```

## 10. 부가 효과 — SEO 시너지

- AI 검색에서 "헬리오시티 전세낀 매물 현황"을 물어볼 때 이 페이지가 인용될 수 있음
